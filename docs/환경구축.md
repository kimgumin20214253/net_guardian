# 우분투 리눅스 환경 구축
- 1단계: 가상화 및 OS 설치 (VMware & Ubuntu)

- 2단계: 시스템 업데이트 및 필수 네트워크 도구 설치

- 3단계: 파이썬 가상 환경(venv) 구축

- 4단계: 프로젝트 필수 파이썬 패키지 설치

- 5단계: 터미널 한글 입력기 설정 (ibus-hangul)

- 6단계: 멀티 터미널 및 파일 구조 개요

## 1단계: 가상화 및 OS 설치 (VMware & Ubuntu)

윈도우 호스트 PC 환경 위에 독립된 리눅스 로컬 실험 환경을 조성하기 위해 가상 컴퓨터를 생성하고 OS 설치
가상 머신 구축: VMware Workstation Player를 설치합니다.

- 아키텍처 선택: 일반적인 윈도우 PC(Intel 또는 AMD CPU) 환경에 맞춰 Intel or AMD 64-bit architecture (x86_64 / amd64) 버전을 선택합니다.

- OS 디스크 이미지: ubuntu-24.04-desktop-amd64.iso (Desktop 버전) 파일을 활용합니다.

- 가상 머신 사양 설정

1) 디스크 용량: 데이터 축적을 위해 최소 25GB ~ 40GB 이상 할당합니다.

2) 메모리(RAM): 최소 2GB(2048MB) ~ 4GB(4096MB)로 상향 조절합니다.

3) 프로세서(CPU): Cores 값을 2 이상으로 설정합니다.

4) 설치 유형: 가상 머신 내부의 가짜 디스크 영역만 포맷하므로, [Erase disk and install Ubuntu] (디스크를 지우고 우분투 설치)를 선택하여 자동 최적화 설치를 진행합니다.



## 2단계: 시스템 업데이트 및 필수 네트워크 도구 설치
우분투 부팅이 완료되면 Ctrl + Alt + T를 눌러 터미널 창을 열고, 패키지 저장소 최신화 및 트래픽 제어(tc), 포트 확인(netstat)을 위한 네트워크 필수 도구를 설치합니다.



- 패키지 목록 업데이트 및 시스템 업그레이드
```
sudo apt update && sudo apt upgrade -y
```
- 필수 네트워킹 및 트래픽 제어(tc) 도구 설치
```
sudo apt install iproute2 net-tools -y
```


## 3단계: 파이썬 가상 환경(venv) 구축
우분투 23.04 버전부터 도입된 외부 패키지 설치 차단 정책(PEP 668 오류)을 우회하고, 프로젝트 라이브러리 간 꼬임 현상을 방지하기 위해 독립된 파이썬 가상 환경을 구축합니다.


- 가상 환경 구축을 위한 파이썬 도구 설치
```
sudo apt install -y python3-venv python3-pip
```   

- venv라는 이름의 가상 환경 생성
```
python3 -m venv venv
```

-  가상 환경 활성화: 활성화가 성공하면 터미널 프롬프트 맨 앞에 (venv)가 표시됩니다.
```
source venv/bin/activate
```

## 4단계: 프로젝트 필수 파이썬 패키지 설치
가상 환경이 활성화된 상태((venv))에서 프로젝트의 핵심 인프라 구현 및 향후 AI 모델 고도화, 반응형 대시보드 구현에 필요한 라이브러리들을 일괄 설치합니다.


- 핵심 패키지 한 번에 설치하기: numpy, pandas, pymodbus, python-dateutil, six ...

```
pip install numpy==2.4.4 pandas==3.0.2 pymodbus==3.1.0 python-dateutil==2.9.0.post0 six==1.17.0
```

## 5단계: 터미널 한글 입력기 설정 (ibus-hangul)

- 한글 입력기 패키지 설치
```
sudo apt install ibus-hangul -y
```

- 우분투 GUI 환경 설정법   
1) 우분투 상단 시스템 설정 혹은 메뉴에서 [Settings (설정)] ➡️ [Keyboard (키보드)] 메뉴로 이동합니다.

2) Input Sources (입력 소스) 항목 아래의 [+] 버튼을 클릭합니다.

3) 리스트에서 Korean이 아니라 반드시 Korean (Hangul)을 선택하여 추가합니다.

4) 추가된 항목 우측의 톱니바퀴 아이콘을 눌러 Hangul Toggle Key 항목에 실제 키보드의 [한/영] 키를 등록해주면 세팅이 완료됩니다.

## 6단계: 멀티 터미널 및 파일 구조 개요

- 터미널 1 (서버 가동): python3 network/server.py ➡️ Modbus TCP 대기 (현장 장비 역할)

- 터미널 2 (데이터 수집): python3 network/client.py ➡️ 시나리오별 RTT/Loss 유기적 데이터 수집

- 터미널 3 (로그 확인): tail -f data/raw_normal.csv ➡️ 데이터 누적 상태 리눅스 표준 실시간 감시
