# 프로젝트 구동 방법
## 공통 (가상환경 활성화)
- 터미널 실행시 먼저 반드시 입력해야 할 터미널 명령어
```
cd ~/바탕화면/Net_Guardian
source venv/bin/activate
```

## 모드버스 TCP 실행 (서버) (1번 터미널)

```
python3 network/server.py
```


## 시나리오별 네트워크 데이터 수집 (클라이언트) (2번 터미널)
- 정상 상태, 지연 발생, 패킷 손실, 복합(지연+패킷) 상황 총 4가지의 시나리오 트래픽을 주입하며 로그 적재
```
python3 network/client.py
```
- 수집 결과 저장 위치
1) 정상 상태: data/raw_normal.csv
2) 지연 상태: data/raw_delay.csv
3) 패킷 손실 상태: data/raw_loss.csv
4) 복합 상태: data/raw_delay_loss.csv


## 인위적인 장애 상황 주입 (2번 터미널)
- 정상적인 데이터 수집
```
sudo tc qdisc del dev lo root 2>/dev/null
python3 network/client.py # 1번 선택

```
- 지연 데이터 수집 
```
sudo tc qdisc add dev lo root netem delay 100ms
python3 network/client.py # 2번 선택
sudo tc qdisc del dev lo root 
```
- 패킷 손실 데이터 수집 
```
sudo tc qdisc add dev lo root netem loss 10%
python3 network/client.py # 3번 선택
sudo tc qdisc del dev lo root 
```
- 복합 데이터 수집 
```
sudo tc qdisc add dev lo root netem delay 100ms loss 10%
python3 network/client.py # 4번 선택
sudo tc qdisc del dev lo root 
```


## 리눅스 표준 실시간 로그 모니터링 (3번 터미널)
- 클라이언트가 파일에 데이터를 정상적으로 쓰고 있는지 실시간으로 감시
- 실시간으로 누적되는 CSV 파일의 하단 텍스트 추적 모니터링
```
tail -f data/raw_normal.csv
tail -f data/raw_delay.csv
tail -f data/raw_loss.csv
tail -f data/raw_delay_loss.csv
```
